## Ветки. Локальная работа с ветками в git.

### Что такое ветки и для чего они нужны при работе с системой контроля версий.

### Базовая работа с ветками в git.

|Название  |  Описание   |Примичание|
|--------|--------|----------|
|**git branch**| выводит список сущетствующих веток| для создания ветки пишем ее название после brunch|
|**git branch --all**| выводит список совсем всех веток|
|**git log --all -p**| покажет всю историю коммитов со всеми подробностьями, включая все изменения в каждом коммите|
|**git -b "yourBranch_name"**|создать и переключиться на ветку yourBranch_name|
|**git checkout "название ветки"**|переход между ветками
|**git branch -d "название ветки"**| удалить ветку
|**git merge "название ветки"**| сливает текушую ветку и указанную ветку|
|**git rebase <название ветки назначения>**| переносит изменения из текущей ветки в названную| написано так Хочется сдвинуть наши изменения из bugFix прямо на вершину ветки main. Благодаря этому всё будет выглядеть, как будто эти изменения делались последовательно, хотя на самом деле - параллельно.|
|**git checkout HEAD^**| перейти на предидущий коммит относительно HEAD|
|**git checkout <название ветки>^**| перейти на предидущий коммит этой ветки|
|**git checkout <название ветки>^^**| перейти на прородителя ветки...|
|**git checkout HEAD~<число>**| перейти на <число> комитов от HEAD|
| **HEAD detached**| после такой навигации голова не на вершине ветки куда мы передем, на коммите куда мы попали в результате навигации. чтобы исправить нужно перейти в ветку где мы находимся| git checkout эта команда переключит проект в последний коммит текущей ветки. Точнее, в состоянии detached head, проект не находится ни в какой ветке. git checkout branchname просто переключит проект на последний коммит той ветки, которая указана|
|**HEAD detached**| Самый правильный вариант **git checkout HEAD@{1}**|
|**git reflog**|Посмотреть где мы находимся|
|**git branch -f <имя_ветки> HEAD~<число>**| Переместить принудительно ветку <имя_ветки> на <число> коммитов назад от HEAD|
|**git log --graph**| показывает в виде дерева, можно проследить когда появлялись и когда сливались|
|**git reset HEAD~<число>**| отменяет изменения, перенося ссылку на ветку назад, на более старый коммит. Это своего рода "переписывание истории"; git reset перенесёт ветку назад, как будто некоторых коммитов вовсе и не было| **только для локальных репозиториев**|
|**git revert HEAD**| Чтобы отменить изменения и поделиться отменёнными изменениями с остальными. Новый коммит просто содержит изменения, полностью противоположные тем, что сделаны в коммите на котором голова.| **для глобальных репозиториев**|
||**Что происходит после таких перемещений**|
|**HEAD detached**| Это может означать, что вы делали какую-то навигацию по истории коммитов, и неправильно вернулись к последнему коммиту. |git checkout - эта команда переключит проект в последний коммит текущей ветки. Точнее, в состоянии detached head, проект не находится ни в какой ветке. git checkout branchname просто переключит проект на последний коммит той ветки, которая указана.|
|**git reflog**| Показывает где мы находимся|
|**git cherry-pick**| Скопировать несколько коммитов на то место, где сейчас находиться HEAD| синтаксис **git cherry-pick <Commit1> <Commit2><...>**|
